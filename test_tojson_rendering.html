Test Jinja2 tojson Rendering in HTML Attributes

Problem: Does tojson work correctly in onclick attributes?

Hypothesis

When Jinja2 renders {{ c.title | tojson }} inside an onclick attribute, if the title contains special characters, the HTML might break.

Test Cases:

1. Simple Text


  


  

    
2. With Apostrophe (Problem case!)

    
    

  


  

    
3. With Quotes

    
    

  


  

    
4. What tojson Should Render (Escaped JSON)

    
    
    
Generated HTML with tojson should look like:
<button onclick="window.startDraw(1, "Sorteio d'Ano")">

But Jinja2 with tojson renders as JSON string:
{{ "" | tojson }} → "" (with quotes as part of string)
    

  


  

    
HTML Source Analysis

    
Right-click → View Source, look for the onclick attribute

    
The actual HTML generated will show if tojson is working correctly

  


  

  


  
Solution:

  
If onclick breaks, the fix is to change the button from:

  
<button onclick="window.startDraw({{ c.id }}, {{ c.title | tojson }})">

  
To use data attributes instead:

  
<button data-draw-id="{{ c.id }}" data-draw-title="{{ c.title | tojson }}" onclick="window.startDraw(this.dataset.drawId, JSON.parse(this.dataset.drawTitle))">

Or use a safer approach:
<button onclick="window.startDraw({{ c.id }}, JSON.parse('{{ c.title | tojson }}'))">
