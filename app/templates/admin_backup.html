🔧 Como configurar upload por link (sem API)

  1. Crie um endpoint de upload e obtenha a URL compartilhada (ex: webhook n8n, Apps Script Web App, URL assinada de storage).
  2. No .env, configure:
BACKUP_SHARED_UPLOAD_URL=https://seu-endpoint/upload
BACKUP_SHARED_UPLOAD_METHOD=PUT
# opcional
BACKUP_SHARED_BEARER_TOKEN=token-secreto

O sistema envia o arquivo em binário com cabeçalhos X-Backup-Filename e X-Backup-Source.

🔧 Modo legado: Google Drive API (opcional)

  1. Acesse o Google Cloud Console e crie um projeto.
  2. Ative a Google Drive API no projeto.
  3. Vá em IAM & Admin → Service Accounts → Criar conta de serviço.
  4. Gere uma chave JSON para a conta → faça download do arquivo.
  5. No Google Drive, crie uma pasta para os backups e compartilhe-a com o e-mail da conta de serviço (ex: backup@meu-projeto.iam.gserviceaccount.com).
  6. Copie o ID da pasta (parte da URL após /folders/).
  7. Adicione ao .env:
# Opção A — caminho para o arquivo JSON (dentro do container)
GDRIVE_CREDENTIALS_JSON=/data/gdrive-credentials.json
GDRIVE_FOLDER_ID=1AbCdEfGhIjKlMnOpQrStUvWxYz0123456

# Opção B — conteúdo JSON direto (útil para variáveis de ambiente em Cloud)
GDRIVE_CREDENTIALS_JSON={"type":"service_account","project_id":"...","private_key":"...","client_email":"..."}

💡 Se usar a Opção A, monte o arquivo no docker-compose.yml: volumes: - ./gdrive-credentials.json:/data/gdrive-credentials.json:ro
